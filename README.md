# nft-dapp-project
NFT Minting DApp for University Project
Чтобы изменить код или файлы прямо в GitHub через браузер, тебе не нужны никакие сторонние программы. Вот подробная инструкция для самых частых действий:

1. Как изменить текст внутри файла (например, вставить адрес контракта в app.js)
Это то, что тебе придется делать чаще всего.

Открой свой репозиторий на GitHub.

Зайди в нужную папку (например, frontend/).

Нажми на имя файла, который хочешь изменить (например, app.js).

В правом верхнем углу окна с кодом нажми на иконку карандаша (при наведении появится надпись Edit this file).

Теперь ты можешь печатать прямо в браузере. Найди строку const CONTRACT_ADDRESS = "..." и вставь туда адрес.

Когда закончишь, нажми на большую зеленую кнопку Commit changes... в правом верхнем углу.

В появившемся окне еще раз нажми Commit changes. Готово!

2. Как полностью заменить файл на новый (если ты что-то переписал на компе)
Если тебе проще работать в блокноте на компьютере, а потом просто «обновить» файл в GitHub:

Зайди в папку на GitHub, где лежит старый файл (например, в frontend/).

Нажми кнопку Add file -> Upload files.

Перетащи новый файл с тем же именем (например, app.js) прямо в окно браузера. GitHub поймет, что это новая версия того же файла.

Нажми Commit changes.

3. Как изменить структуру (удалить лишнее или переименовать)
На твоем скрине видно, что файлы .gitkeep лежат «в куче». Чтобы их почистить:

Чтобы удалить: нажми на файл, затем на иконку корзины (рядом с карандашом) и подтверди через Commit changes.

Чтобы переименовать: нажми на карандаш (редактирование) и просто измени имя файла в поле вверху страницы.

Что тебе нужно изменить прямо сейчас (судя по твоим скринам):
Судя по файлу на скрине, твой проект не заработает, пока ты не сделаешь следующее:

Открой frontend/app.js через карандаш.

В строке const CONTRACT_ADDRESS замени "PASTE_YOUR_CONTRACT_ADDRESS_HERE" на реальный адрес из Remix.

В массив const CONTRACT_ABI = [...] вставь ABI, который скопируешь в Remix после деплоя.

Важный совет: Если ты все-таки решишь залить всё одним архивом (ZIP), GitHub его не «распакует» сам. Люди увидят просто один файл project.zip. Чтобы проект открывался как сайт, файлы должны лежать россыпью, как на твоем втором скрине.
